## 概述 (Overview)

- **数据库版本**：MySQL 8.0
- **字符集**：utf8mb4
- **排序规则**：utf8mb4_unicode_ci
- **存储引擎**：InnoDB

## 通用约定

- **主键**：`id VARCHAR(32) NOT NULL`，PRIMARY KEY，**雪花算法**生成。所有跨表关联列（`claim_id`/`biz_id`/`attachment_ids` 等）同为 `VARCHAR(32)`，**逻辑关联、不建物理外键**。
- **审计列（每表统一，SQL 里放表末，本文档各表不再重复列出）**：

| 字段名 | 数据类型 | 长度 | 允许NULL | 默认值 | 约束 | 说明 |
|--------|---------|------|---------|--------|------|------|
| create_by | VARCHAR | 64 | YES | NULL | - | 创建人 |
| created_at | DATETIME | - | NO | CURRENT_TIMESTAMP | - | 创建时间 |
| update_by | VARCHAR | 64 | YES | NULL | - | 修改人 |
| updated_at | DATETIME | - | NO | CURRENT_TIMESTAMP | ON UPDATE | 更新时间 |
| deleted_at | DATETIME | - | YES | NULL | - | 软删除时间戳（NULL=未删） |
| version | INT | - | NO | 0 | - | 乐观锁版本号 |

- **金额**：`DECIMAL(18,2)`，可正可负者注明（不加 UNSIGNED）；币种本期仅 `CNY`（`USD` 保留禁用，A-29）。
- **命名**：表名 / 列名 snake_case 小写。**每表 `COMMENT` 表注释、每列 `COMMENT` 字段注释**，枚举列注释列全码值语义（见枚举字典）。

### 枚举字典

| 字典 | 码值（code=中文语义） |
|---|---|
| `CLAIM_STATUS` 服务商索赔单状态 | DRAFT=草稿 / PENDING_REGION=待区域初审 / PENDING_HQ=待总部审核 / PENDING_LIABILITY=待责任判定 / PENDING_SETTLE=待结算 / REJECTED=已驳回 / DECLINED=已拒绝 / SETTLED=已结算 / VOIDED=已作废 |
| `CLAIM_TYPE` 索赔类型 | WARRANTY=保内维修 / SERVICE_ACTIVITY=服务活动 / INTERNAL=内部服务 / SPECIAL=特殊工单 |
| `LIABILITY_RESULT` 责任判定结果 | SUPPLIER=供应商责任 / PROVIDER=服务商责任 / INTERNAL=内部责任 |
| `CLAIM_TARGET` 索赔对象 | XW=欣旺达件 / SELF=自采件 |
| `CURRENCY` 币种 | CNY=人民币 / USD=美元(本期禁用) |
| `DEDUCT_STATUS` 抵扣单状态 | UNSETTLED=未结算 / SETTLED=已纳入结算 |

### 单据编码

`<前缀>-yyyyMMdd-4位日内流水`，全局唯一（唯一索引 + 号段/序列并发控制，D2 落实现）。

| 单据 | 前缀 | 示例 |
|---|---|---|
| 服务商索赔单 | `SPCO` | SPCO-20260712-0007 |
| 索赔抵扣单 | `DKD` | DKD-20260620-0006 |

## 字段说明 (Field Descriptions)

### 核心字段
- **id**：VARCHAR(32) 雪花算法主键；跨表关联列同为 VARCHAR(32)，逻辑关联不建物理外键。
- **单据编号**（claim_no/deduct_no…）：`前缀-yyyyMMdd-4位流水`，全局唯一，见单据编码。
- **work_order_id / work_order_no / provider_id / main_part_id**：来自既有工单/服务商模块，**本期 Mock、待挂载**，物理绑定后回填。
- **status**：一律引用枚举字典码值，禁止枚举外取值；按钮/流转由 Service 状态机控制。

### 金额与快照
- **金额列** DECIMAL(18,2)；可正可负的列不加 UNSIGNED，具体见各表列注释。
- **coefficient_snapshot**：索赔发起时点的服务商系数快照，落单存储，系数变更不影响历史（A-2）。
- **备件管理费** = 备件销售价 × 系数；**申请总金额** = 工时费 + 备件管理费 + 其它费用（自动，只读）。
- **抵扣金额** = 服务商责任对应索赔金额 × 责任承担比例（A-30）。

### 软删除与乐观锁
- **deleted_at**：软删除时间戳，NULL=未删；所有业务查询附 `deleted_at IS NULL`。
- **version**：乐观锁版本号，核心单据（如索赔单）更新时 `WHERE version=? ` 并 `version=version+1`，实现并发控制（PRD S10.2）。

## 常用查询示例 (Common Queries)

```sql
-- 1. 新建服务商索赔单（草稿）
INSERT INTO claim_order
  (id, claim_no, work_order_id, work_order_no, provider_id, provider_code, provider_name,
   region, claim_type, labor_fee_total, part_mgmt_fee, other_fee_total, apply_total_amount,
   currency, status, coefficient_snapshot, create_by, created_at)
VALUES
  (:id, 'SPCO-20260712-0007', :woId, :woNo, :providerId, :providerCode, :providerName,
   '华东区/江苏', 'WARRANTY', 600.00, 3120.00, 140.00, 3860.00,
   'CNY', 'DRAFT', 1.2500, :operator, NOW());

-- 2. 按单号查（未删）
SELECT * FROM claim_order WHERE claim_no = 'SPCO-20260712-0007' AND deleted_at IS NULL;

-- 3. 作废草稿索赔单（保留业务数据供全状态查看，不做软删除）
UPDATE claim_order SET status = 'VOIDED', update_by = :operator,
    version = version + 1, updated_at = NOW()
WHERE id = :id AND status = 'DRAFT' AND version = :version AND deleted_at IS NULL;
```
