## 概述 (Overview)

- **数据库版本**：MySQL 8.0
- **字符集**：utf8mb4
- **排序规则**：utf8mb4_unicode_ci
- **存储引擎**：InnoDB

## 通用约定

- **主键**：`id VARCHAR(32) NOT NULL`，PRIMARY KEY，**雪花算法**生成。所有跨表关联列（`claim_id`/`settle_id`/`xw_claim_id`/`account_id`/`payment_id`/`*_attachment_id` 等）同为 `VARCHAR(32)`，**逻辑关联、不建物理外键**。
- **审计列（每表统一，SQL 里放表末，本文档各表不再重复列出）**：

| 字段名 | 数据类型 | 长度 | 允许NULL | 默认值 | 约束 | 说明 |
|--------|---------|------|---------|--------|------|------|
| create_by | VARCHAR | 64 | YES | NULL | - | 创建人 |
| created_at | DATETIME | - | NO | CURRENT_TIMESTAMP | - | 创建时间 |
| update_by | VARCHAR | 64 | YES | NULL | - | 修改人 |
| updated_at | DATETIME | - | NO | CURRENT_TIMESTAMP | ON UPDATE | 更新时间 |
| deleted_at | DATETIME | - | YES | NULL | - | 软删除时间戳（NULL=未删） |
| version | INT | - | NO | 0 | - | 乐观锁版本号 |

- **金额**：`DECIMAL(18,2)`，可正可负者注明（不加 UNSIGNED）；币种本期仅 `CNY`（`USD` 保留禁用，A-29）。
- **命名**：表名 / 列名 snake_case 小写。**每表 `COMMENT` 表注释、每列 `COMMENT` 字段注释**，枚举列注释列全码值语义（见枚举字典）。

### 枚举字典

| 字典 | 码值（code=中文语义） |
|---|---|
| `CLAIM_STATUS` 服务商索赔单状态 | DRAFT=草稿 / PENDING_REGION=待区域初审 / PENDING_HQ=待总部审核 / PENDING_LIABILITY=待责任判定 / PENDING_SETTLE=待结算 / REJECTED=已驳回 / DECLINED=已拒绝 / SETTLED=已结算 / VOIDED=已作废 |
| `XW_CLAIM_STATUS` 闪欣索赔单状态 | DRAFT=草稿 / PENDING_AUDIT=待审核 / PENDING_SETTLE=待结算 / REJECTED=已驳回 / DECLINED=已拒绝 / SETTLED=已结算 / VOIDED=已作废 |
| `SETTLE_STATUS` 结算单状态 | PENDING_CONFIRM=待确认 / PENDING_RECON=待对账 / PENDING_INVOICE=待开票 / PENDING_VERIFY=待核验 / VERIFIED=已核验 / PENDING_FORCE=待强制结算 / SETTLED=已结算 / VOIDED=已作废 |
| `PAYMENT_STATUS` 付款申请单状态 | PENDING_SUBMIT=待提交 / APPROVING=审批中 / APPROVED=审批通过 / REJECTED=审批驳回（兼容码，本期不驻留，驳回直接回 PENDING_SUBMIT）/ PAID=已付款 / VOIDED=已作废【D-付款-D1】 |
| `CLAIM_TYPE` 索赔类型 | WARRANTY=保内维修 / SERVICE_ACTIVITY=服务活动 / INTERNAL=内部服务 / SPECIAL=特殊工单 |
| `LIABILITY_RESULT` 责任判定结果 | SUPPLIER=供应商责任 / PROVIDER=服务商责任 / INTERNAL=内部责任 |
| `SETTLE_TYPE` 结算类型 | PROVIDER=服务商结算 / SHANXIN=闪欣动力结算 |
| `CLAIM_TARGET` 索赔对象 | XW=欣旺达件 / SELF=自采件 |
| `FEE_OWNER` 费用归属 | XW=欣旺达 / SHANXIN=闪欣动力 / INTERNAL_DEPT=内部部门 |
| `XW_AUDIT_RESULT` 欣旺达审核结果 | PASS=通过 / REJECT=驳回 / DECLINE=拒绝 / PARTIAL=部分通过 |
| `PAY_GRADE` 付款金额档 | L1=<5万 / L2=5万~20万 / L3=≥20万 |
| `CURRENCY` 币种 | CNY=人民币 / USD=美元(本期禁用) |
| `DEDUCT_STATUS` 抵扣单状态 | UNSETTLED=未结算 / SETTLED=已纳入结算 |

### 单据编码

`<前缀>-yyyyMMdd-4位日内流水`，全局唯一（唯一索引 + 号段/序列并发控制，D2 落实现）。

| 单据 | 前缀 | 示例 |
|---|---|---|
| 服务商索赔单 | `SPCO` | SPCO-20260712-0007 |
| 闪欣向欣旺达索赔单 | `SXCO` | SXCO-20260712-0003 |
| 服务商结算单 | `SPJS` | SPJS-20260701-0012 |
| 闪欣结算单 | `SXJS` | SXJS-20260701-0005 |
| 付款申请单 | `JSFK` | JSFK-20260702-0012 |
| 索赔抵扣单 | `DKD` | DKD-20260620-0006 |

## 字段说明 (Field Descriptions)

### 核心字段
- **id**：VARCHAR(32) 雪花算法主键；跨表关联列同为 VARCHAR(32)，逻辑关联不建物理外键。
- **单据编号**（claim_no/settle_no/apply_no/deduct_no…）：`前缀-yyyyMMdd-4位流水`，全局唯一，见单据编码。
- **work_order_id / work_order_no / provider_id / main_part_id**：来自既有工单/服务商模块，**本期 Mock、待挂载**，物理绑定后回填。
- **status**：一律引用枚举字典码值，禁止枚举外取值；按钮/流转由 Service 状态机控制。

### 金额与快照
- **金额列** DECIMAL(18,2)；`payable_amount`、`warranty_deposit_txn.amount`、`claim_settle_detail.deduct_amount` 可正可负。
- **coefficient_snapshot**：索赔发起时点的服务商系数快照，落单存储，系数变更不影响历史（A-2）。
- **备件管理费** = 备件销售价 × 系数；**申请总金额** = 工时费 + 备件管理费 + 其它费用（自动，只读）。
- **抵扣金额** = 服务商责任对应索赔金额 × 责任承担比例（A-30）。

### 软删除与乐观锁
- **deleted_at**：软删除时间戳，NULL=未删；所有业务查询附 `deleted_at IS NULL`。
- **version**：乐观锁版本号，核心单据（索赔/结算/付款/质保金）更新时 `WHERE version=? ` 并 `version=version+1`，实现并发控制（PRD S10.2）。

## 常用查询示例 (Common Queries)

```sql
-- 1. 新建服务商索赔单（草稿）
INSERT INTO claim_order
  (id, claim_no, work_order_id, work_order_no, provider_id, provider_code, provider_name,
   region, claim_type, labor_fee_total, part_mgmt_fee, other_fee_total, apply_total_amount,
   currency, status, coefficient_snapshot, create_by, created_at)
VALUES
  (:id, 'SPCO-20260712-0007', :woId, :woNo, :providerId, :providerCode, :providerName,
   '华东区/江苏', 'WARRANTY', 600.00, 3120.00, 140.00, 3860.00,
   'CNY', 'DRAFT', 1.2500, :operator, NOW());

-- 2. 按单号查（未删）
SELECT * FROM claim_order WHERE claim_no = 'SPCO-20260712-0007' AND deleted_at IS NULL;

-- 3. 结算单生成：按服务商+周期汇总业务审核通过未结算索赔单
-- :periodEndExclusive = 存储的结算周期止日 + 1 日，避免漏截止日当天
SELECT c.provider_id, SUM(c.apply_total_amount) AS claim_payable, COUNT(*) AS claim_count
FROM claim_order c
JOIN claim_audit_record ar
  ON ar.biz_type = 'CLAIM' AND ar.biz_id = c.id
 AND ar.audit_stage = '业务审核' AND ar.audit_result = '通过' AND ar.deleted_at IS NULL
WHERE c.status = 'PENDING_SETTLE' AND c.deleted_at IS NULL
  AND ar.audit_time >= :periodStart AND ar.audit_time < :periodEndExclusive
GROUP BY c.provider_id
HAVING SUM(c.apply_total_amount) IS NOT NULL;

-- 4. 乐观锁更新结算单状态（确认结算→待对账）
UPDATE claim_settle_order
SET status = 'PENDING_RECON', recon_confirm_time = NULL, update_by = :operator,
    version = version + 1, updated_at = NOW()
WHERE id = :id AND status = 'PENDING_CONFIRM' AND version = :version AND deleted_at IS NULL;

-- 5. 作废草稿索赔单（保留业务数据供全状态查看，不做软删除）
UPDATE claim_order SET status = 'VOIDED', update_by = :operator,
    version = version + 1, updated_at = NOW()
WHERE id = :id AND status = 'DRAFT' AND version = :version AND deleted_at IS NULL;

-- 6. 质保金扣减流水入账（负数结算强制结算）
INSERT INTO warranty_deposit_txn
  (id, account_id, provider_id, biz_type, biz_no, amount, balance_after, shortfall_carryover,
   voucher_attachment_id, create_by, created_at)
VALUES
  (:id, :accountId, :providerId, 'DEDUCT', 'SPJS-20260701-0014', -3200.00, 0.00, 700.00,
   :voucherId, :operator, NOW());   -- 余额不足扣全部、差额 700 挂账结转
```
