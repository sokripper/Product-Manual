# 原型说明与生产前端交接（B2）

> 形态：**HTML 原型（SDD 模式 A，默认）**，可在浏览器直接打开、可点击、可截图。仅用于设计验证，不作生产前端源码。字段/状态真相以 `docs/PRD.md` S4/S7 为准。
> 打开方式：直接开 `index.html`（总览，含全部页面入口），或起本地静态服务 `python3 -m http.server` 后访问 `pages/*.html`（file:// 下部分浏览器会拦本地脚本）。


## 三、设计验证结论

✅ 全部 12 页对照 `.sdd/tmp/ui-design-spec.md` 组件清单一致：AntD 蓝设计系统、高密度表格+合计行、九态中文 Tag、审批流时间线(签名交互)、状态色映射。已在浏览器 1440 宽验证 P01/P02(+弹窗)/P10 渲染正常、无控制台报错。
✅ **域内容已修正**：全部为保内维修索赔域（工时费/备件管理费/维修工单/SPCO·SXCO·SPJS·JSFK 单号/九态中文），取代首版 Stitch 的物流理赔占位。
✅ 40 条决策落地：待责任判定态、待强制结算态、付款已作废态、责任抵扣列、部分通过核准金额、白名单=开票权限、系数按服务商快照、财务共享/OA 分账提示等均已在对应页面体现。

## 四、生产前端交接

- **关键组件**：左导航+面包屑、筛选区、数据表格(排序/分页/行操作/合计行)、状态 Tag、详情卡片组、明细表(含抵扣/合计)、审批流时间线、弹窗组、底部固定操作栏。样式 token 见 `assets/style.css` `:root`。
- **交互状态**：按钮显示严格按「状态×角色」（各页操作列/底部栏已注明条件），实现以 PRD S7 为准。
- **Mock 数据**：原型为静态示例；真实数据结构以 `docs/PRD.md` S10 + 阶段 D 表结构文档为准。
- **需真实 API**：所有列表查询/详情/审核/结算/开票/付款/配置；接口以阶段 E `docs/api-contracts/接口文档-服务商索赔单.md` 为准。
- **必须生产实现的视觉**：状态色映射、审批流时间线、金额 mono 对齐、合计行；**仅原型展示**：示例数据/文案。
- **视口**：桌面 1440×900（B 端后台，无移动端要求）。
