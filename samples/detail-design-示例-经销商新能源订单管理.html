3. 状态机

3.1 订单状态（T_V_Z_BU_DLR_ORDER.STATE）

  • 状态全集：新建 / 待审批 / 部分待审批 / 部分完成 / 完成 / 拒绝 / 作废
当前态事件目标态前置条件触发副作用
—代新增保存新建—4.2
新建代作废作废仅新建可作废4.4
新建代提报(全匹配)待审批全部单车匹配到资源4.5-A
新建代提报(部分匹配)部分待审批部分匹配4.5-B
新建代提报(无匹配)新建(不变)无资源4.5-C
待审批审批(全通过)完成全行待审批且无拒绝行4.6-A
待审批/部分待审批/部分完成审批(部分通过)部分完成部分行通过4.6-B
待审批/部分待审批拒绝(全部)拒绝勾选全部待审批行4.7-A
待审批/部分待审批/部分完成拒绝(部分)存在完成行→部分完成；否则部分待审批勾选部分待审批行4.7-B
  • 非法迁移拦截：非"新建"不可代修改/代作废/代提报；勾选含 拒绝/完成 行时 审核/拒绝 按钮置灰。

3.2 订单行状态（T_V_Z_BU_DLR_ORDER_D.DLR_ORDER_D_STATUS）

  • 状态全集：新建 / 待审批 / 完成 / 拒绝 / 作废
  • 主/行联动：全部行完成→单据完成；部分行完成+存在拒绝行→部分完成；全部行拒绝→单据拒绝。

4.5 操作：代提报 ★（三分支五表——模板核心压测点）

  • 可用条件：仅 新建；二次确认「您确认要提报选择的数据吗？」
  • 匹配逻辑（只匹配现车）：中心库 T_M_Z_BD_CAR（CAR_STATE∈{2,12} && ONECAR_FLAG(No/null) && WAREHOUSE_CODE='A' && EXPECT_SUB_WAREHOUSE=null && LOGISTICS_CAR_FLAG=Yes && IS_SALE_CLOSED=Yes && 整车共享经销商=null && OFFLINE_DATE≠null）JOIN VLS_T_WH_VIN（ISDISTRIBUTION=未分车 && VLS_STATE=已入库 && CAR_QUALITY_STATUS=正常 && IS_LOGISTICS_SEPARABLE=否 && ISMODEL=否 && ZEBRASTATE=否 && ISLEND∈{空,归还} && INV_FROZEN=否），按 OFFLINE_DATE 升序占用。

分支 B：部分匹配 → 订单=部分待审批

  • 主单 STATE=部分待审批（其余同上）；已匹配行按分支 A 的 4 张表写入；未匹配行：T_V_Z_BU_DLR_ORDER_D UPDATE 行状态=拒绝、修改时间/人。

分支 C：无匹配

  • 不更新任何数据，提示「系统可用资源不足！」
  • 事务边界：一次代提报的全部表写入必须在单一事务内；任一表失败整单回滚，车辆资源不得半占用。
  • 结果反馈：成功返回主界面并刷新状态。

4.6 操作：审批（要点，勾选相关分支）

  • 可用条件：待审批 / 部分待审批 / 部分完成；勾选含 拒绝/完成 行时审核置灰。
  • 分支 A（勾选全部待审批行且无拒绝行）：INSERT T_V_Z_BU_CAR_APPROVE(APPROVE_CODE=交付中心编码+EO+YYYYMMDD+四位流水，审批状态=有效)、INSERT T_V_Z_BU_CAR_APPROVE_HIS(类型=审批通过，子单号=审批单号+三位流水)、UPDATE T_M_Z_BD_CAR(分车时间=审批时间)、INSERT VLS_T_IF_D2L_CARSPLIT_RESULT、UPDATE T_V_Z_BU_DLR_ORDER_D(行状态=完成+回填审批单号/子单号/行状态/创建修改人时间+分车时间)、UPDATE 主单 STATE=完成。
  • 分支 B（勾选部分待审批行，或全选但含拒绝行）：同上但只作用待审批行；主单 STATE=部分完成。

4.7 操作：拒绝 ★（逆向回滚——见第 6 节）

  • 可用条件：仅可勾选 待审批 行（含 拒绝/完成 则置灰）。
  • 分支 A（勾选全部待审批行）→ 主单 STATE=拒绝；分支 B（部分）→ 存在完成行则 部分完成，否则 部分待审批。副作用见第 6 节逆向表。

6. 并发 / 幂等 / 回滚
